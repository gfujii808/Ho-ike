# Hōʻike Figma Prompt Pack

Use these prompts to generate only the most important mockups before the competition. The goal is not to redesign the product from scratch. The goal is to improve the visual finish of the existing Hōʻike system.

## Prompt 1: Hero + Student Dashboard + Professional Why

Use this first. It is the highest-value screen for judges.

```text
Design one polished web dashboard screen for “Hōʻike Explorer,” a Hawaiʻi-centered, school-safe career navigation tool for HIDOE middle and high school students.

Important: this should feel like a premium student-facing product, not a generic edtech dashboard, social network, or job board. Keep the content grounded in the exact product structure below. Do not invent a different concept.

Goal of this screen:
Create the strongest possible first impression for judges in under 30 seconds.

This one screen should include 3 areas:
1. Hero / first impression
2. Student dashboard
3. Professional Why

Product concept:
Hōʻike helps students understand how assignments, hobbies, interests, and community activities connect to local futures in Hawaiʻi. It “unhides the hidden curriculum” of career navigation by turning what students already do into pathway visibility, school-safe Local Voices, and Small Steps.

Visual style:
- modern, premium, rounded, student-friendly
- flat color blocks, not gradients
- dark green brand base
- earthy maroon accent
- support colors: soft peach, blush, pale yellow, mint, powder blue
- warm off-white / sand background
- strong visual hierarchy
- bold but clean typography
- culturally grounded, hopeful, not corporate and not childish

Typography feel:
- headings similar to Red Hat Display
- body/UI similar to Lexend Deca

Content to include:

HERO
- product name: “Hōʻike Explorer”
- main statement: “Revealing the hidden curriculum for attainable local futures in Hawaiʻi.”
- one short supporting line:
  “Hōʻike turns assignments, hobbies, and community activities into attainable dreams that can be used here in Hawaiʻi.”
- optional small product tags such as:
  - Codex-built browser prototype
  - school-safe exploration
  - Hawaiʻi-centered

STUDENT DASHBOARD
- pseudonymous handle such as SunnyMango or CuriousPueo
- class space such as MangoGrove, PueoPod, or HonuHui
- starter profile chips with interests like:
  - drawing and design
  - music
  - community service
- one large jersey-style number tile for saved ideas / saved pathways

PROFESSIONAL WHY
This should be the most visually important card on the screen.
Include:
- small labels:
  - Live translation demo
  - OpenAI Codex-guided scan
- large headline like:
  “Science becomes place-based stewardship.”
- one short highlight statement
- compact chips such as:
  - field
  - observation
  - pattern recognition
  - care for place
- small stat tiles such as:
  - signals spotted
  - pathways opened

Design constraints:
- keep copy short and highly scannable
- use cards, chips, and strong numbers instead of dense text
- no open networking or social-media feel
- keep the product feeling protected, school-safe, and community-rooted
- make it look more polished than a hackathon demo, but still believable as a browser product
```

## Prompt 2: Pathway Exploration + Local Voices

Use this after the first screen is strong.

```text
Design one polished student workflow screen for Hōʻike Explorer focused on Pathway Exploration and Local Voices.

Important: do not redesign the product from scratch. Use the same visual language as the existing Hōʻike hero screen and keep the product school-safe, Hawaiʻi-centered, and pseudonymous.

This screen should include:
- a clickthrough Pathway Exploration card with left/right arrows
- one pathway visible at a time
- a short pathway title
- a short “Good if” line
- a short “Helps here by” line
- a save button
- compact chips for values / local relevance / different ways in

Possible pathways:
- Environmental Monitoring
- Healthcare Diagnostics
- Agri-Tech and Sensors
- Education and Outreach
- Project and Team Support

Also include:
- a Local Voices card
- one school-safe local profile at a time
- left/right navigation
- role and island/location
- a short quote
- actions like:
  - Use question starter
  - Watch intro
  - Vote to invite

The key feeling should be:
- low-pressure exploration
- local futures feel reachable
- pathways feel culturally grounded, not generic
- professional discovery feels guided and protected, not like social media
```

## Prompt 3: Instructor Dashboard

Use this to polish the teacher/counselor side.

```text
Design one polished instructor dashboard screen for Hōʻike Explorer, a Hawaiʻi-centered, school-safe career navigation tool for HIDOE students.

This screen should feel more like a dashboard than a feed.

Include:
- Classroom Snapshot
- large jersey-style numbers
- short blurbs below each number
- Invite Signals / Talk Story
- Moderation Queue

The screen should clearly show:
- pathway saves
- question drafts
- students ready for follow-up
- most-requested Local Voices
- teacher-managed approval / review actions

Visual direction:
- stronger grid and dashboard rhythm
- premium but approachable
- same product family as the student-facing screens
- no corporate enterprise vibe
- use strong spacing, numbers, and color-blocked cards
```

## Prompt 4: Extension Overlay

Use this last before the competition.

```text
Design one narrow browser-extension overlay mockup for Hōʻike Explorer sitting on top of a neutral Canvas-style assignment page.

Important:
- the host LMS should stay visually neutral
- the Hōʻike overlay should feel warmer, rounder, and more student-friendly
- keep the overlay believable as a real browser sidebar

Include:
- Student view / Instructor view toggle
- Professional Why
- Pathway Exploration
- Local Voices
- Small Steps

Design goals:
- one pathway at a time
- one Local Voices profile at a time
- compact cards
- low text density
- same visual language as the main Hōʻike dashboard
- make it clear this is a school-safe overlay, not an open social tool
```

## Figma Rules

- Generate only `1–2` meaningful passes per screen before deciding whether it is good enough.
- Use Figma to improve hierarchy, rhythm, and visual finish, not to invent new product modules.
- If a Figma result is beautiful but less clear than the live demo, do not port it into code.
- Before April 30, only port the top `2–3` most valuable visual upgrades from Figma into the live demo.
