# Hōʻike Figma + Ship Plan

## Goal

Use Figma as a high-leverage art direction layer between now and the **April 30, 2026** competition deadline, then reuse those same design assets later for a **late Fall / Winter 2026** HIDOE or research-facing pitch.

This plan is intentionally `judge-first`, `balanced-credit`, and `reference-first`:

- `judge-first`: prioritize the screens that most improve competition odds
- `balanced-credit`: use Figma steadily without burning through daily credits on broad exploration
- `reference-first`: use Figma to sharpen the live demo, not replace it

## Ground Rules

- The live main demo remains the primary judged artifact.
- The extension mockup remains a secondary artifact and near-term product direction.
- Figma should not redefine the product model or introduce new core surfaces before the competition.
- All mockups should stay grounded in the current Hōʻike structure:
  - `Professional Why`
  - `Pathway Exploration`
  - `Local Voices`
  - `Small Steps`
  - student dashboard
  - instructor dashboard
  - Canvas overlay / extension direction

## Figma Credit Strategy

Use the daily `150 credit` limit carefully:

- default target: `1 strong generation + 1 refinement + optional 1 cleanup pass`
- stop when a screen is good enough to guide implementation
- do not spend credits designing every state or screen variant
- prioritize depth on 4 key screens over breadth across the whole product

## Priority Screens

### 1. Main hero + student dashboard + Professional Why

This is the highest-value screen for judges and should be tackled first.

Must communicate immediately:

- what Hōʻike is
- what Codex is doing
- what students get

This screen sets:

- top-of-page hierarchy
- dashboard rhythm
- strongest visual treatment for `Professional Why`

### 2. Pathway Exploration + Local Voices

This is the most important student workflow screen after the hero.

It should establish:

- clickthrough pathway styling
- school-safe `Local Voices` profile styling
- short, high-signal card language
- stronger relationship between pathways, guided questions, and next steps

### 3. Instructor dashboard

This screen should refine:

- `Classroom Snapshot`
- invite signals
- moderation queue
- dashboard numbers and blurbs

It should preserve the stronger dashboard energy already emerging in the live demo.

### 4. Extension overlay

This should be the last competition-priority Figma screen.

Focus on:

- narrow, believable Canvas overlay
- same product language as the main demo
- compact, high-signal student cards
- credible instructor view in the extension surface

## Integration Rules Before April 30

Only port the top `2–3` visual improvements from Figma into the live demo.

Default port targets:

- hero / first-impression layout
- `Professional Why` card hierarchy
- student dashboard number-card styling
- pathway / `Local Voices` card styling
- instructor dashboard spacing and number treatment

Do not aim for full parity between Figma and the live demo before submission.

If a Figma concept is beautiful but harms:

- clarity
- speed
- school-safe tone
- Codex legibility

then the live demo should keep the clearer version.
