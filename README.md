# Hōʻike Challenge Package

Hōʻike Explorer is a Codex-built browser prototype for the OpenAI x Handshake Codex Creator Challenge. It is designed for HIDOE middle and high school students and demonstrates how OpenAI Codex can translate school assignments into local pathway exploration, safer professional discovery, and small next steps.

This workspace contains the full submission package:

- `submission/hoike-proposal.md` - the polished proposal
- `submission/hoike-executive-summary.md` - the judge-friendly executive brief
- `submission/hoike-demo-script.md` - a 3-minute finalist demo script
- `submission/handshake-ai-showcase.md` - submission-ready Handshake AI Showcase copy
- `submission/hoike-founder-brief.md` - a founder-style launch memo for judges
- `submission/competition-checklist.md` - packaging checklist for the challenge
- `demo/index.html` - a lightweight MVP walkthrough
- `demo/styles.css` - demo styling
- `demo/app.js` - demo interactions and translation logic
- `AGENTS.md` - project guidance for future Codex iterations

## What Judges Should Notice

- `Execution`: the demo includes a live paste-any-assignment flow and an `AI Translation Log` that makes the Codex-powered reasoning legible even when hosted as a static browser prototype.
- `Usefulness / Value`: the product focuses on a real workflow problem for Hawaiʻi students by reducing informational frictions around pathway exploration, local professional discovery, and safer first questions.
- `Polish / Thoughtfulness`: the prototype includes pseudonymous student handles, moderated question flows, teacher-managed classroom invite signals, and concrete `Do Now / Try This Week / Explore Next` follow-up steps.

## Built With Codex

This project was developed using a vibe-coding workflow: natural-language prompting and iterative guidance with an AI coding assistant to generate, refine, and debug the prototype. That workflow was especially useful for:

- building the browser-based demo flow quickly
- iterating on CSS and interaction details
- wiring assignment translation outputs into pathway, prompt, and progress UI
- making the experience feel native to a student workflow inspired by Canvas

## Demo Notes

- The hosted demo is a browser prototype, not a production Canvas extension.
- The `AI Translation Log` is intentionally visible in the demo so reviewers can see how the assignment-to-pathway reasoning works.
- `Local Voices` profiles are simulated for presentation purposes.

Open `demo/index.html` in a browser to review the MVP flow.
